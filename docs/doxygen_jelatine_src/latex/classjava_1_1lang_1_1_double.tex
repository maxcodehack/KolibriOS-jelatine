NaN and infinity are special cases, to allow parsing of the output of toString. Otherwise, the result is determined by calculating {\itshape n $\ast$ 10$^{\mbox{exponent}}$ \/} to infinite precision, then rounding to the nearest double. Unlike \doxyref{Integer.parseInt}{p.}{classjava_1_1lang_1_1_integer_a369fccaa67bcd1e0f7d6695923628a65}, this does not accept Unicode digits outside the ASCII range.
