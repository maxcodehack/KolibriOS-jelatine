This method reads a {\ttfamily String} from an input stream that is encoded in a modified UTF-\/8 format. This format has a leading two byte sequence that contains the remaining number of bytes to read. This two byte sequence is read using the {\ttfamily \doxyref{readUnsignedShort()}{p.}{interfacejava_1_1io_1_1_data_input_a6d1127e06c7ea968fbab4fc73c4014b8}} method of this interface.

After the number of remaining bytes have been determined, these bytes are read an transformed into {\ttfamily char} values. These {\ttfamily char} values are encoded in the stream using either a one, two, or three byte format. The particular format in use can be determined by examining the first byte read. 

If the first byte has a high order bit of 0, then that character consists on only one byte. This character value consists of seven bits that are at positions 0 through 6 of the byte. As an example, if {\ttfamily byte1} is the byte read from the stream, it would be converted to a {\ttfamily char} like so: 

{\ttfamily (char)byte1} 

If the first byte has 110 as its high order bits, then the character consists of two bytes. The bits that make up the character value are in positions 0 through 4 of the first byte and bit positions 0 through 5 of the second byte. (The second byte should have 10 as its high order bits). These values are in most significant byte first (i.e., \char`\"{}big endian\char`\"{}) order. 

As an example, if {\ttfamily byte1} and {\ttfamily byte2} are the first two bytes read respectively, and the high order bits of them match the patterns which indicate a two byte character encoding, then they would be converted to a Java {\ttfamily char} like so: 

{\ttfamily (char)(((byte1 \& 0x1F) $<$$<$ 6) + (byte2 \& 0x3F))} 

If the first byte has a 1110 as its high order bits, then the character consists of three bytes. The bits that make up the character value are in positions 0 through 3 of the first byte and bit positions 0 through 5 of the other two bytes. (The second and third bytes should have 10 as their high order bits). These values are in most significant byte first (i.e., \char`\"{}big endian\char`\"{}) order. 

As an example, if {\ttfamily byte1}, {\ttfamily byte2}, and {\ttfamily byte3} are the three bytes read, and the high order bits of them match the patterns which indicate a three byte character encoding, then they would be converted to a Java {\ttfamily char} like so:

{\ttfamily  (char)(((byte1 \& 0x0F) $<$$<$ 12) + ((byte2 \& 0x3F) + (byte3 \& 0x3F)) }

Note that all characters are encoded in the method that requires the fewest number of bytes with the exception of the character with the value of {\ttfamily \&lt;llll$>$u0000} which is encoded as two bytes. This is a modification of the UTF standard used to prevent C language style {\ttfamily NUL} values from appearing in the byte stream. 

This method can read data that was written by an object implementing the {\ttfamily writeUTF()} method in {\ttfamily \doxyref{DataOutput}{p.}{interfacejava_1_1io_1_1_data_output}}.
