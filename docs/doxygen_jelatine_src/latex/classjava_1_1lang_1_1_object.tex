\subsection{Detailed Description}
\doxyref{Object}{p.}{classjava_1_1lang_1_1_object} is the ultimate superclass of every class (excepting interfaces). When you define a class that does not extend any other class, it implicitly extends \doxyref{java.lang.Object}{p.}{classjava_1_1lang_1_1_object}. Also, an anonymous class based on an interface will extend \doxyref{Object}{p.}{classjava_1_1lang_1_1_object}.

It provides general-\/purpose methods that every single \doxyref{Object}{p.}{classjava_1_1lang_1_1_object}, regardless of race, sex or creed, implements. All of the public methods may be invoked on arrays or interfaces. The protected methods {\ttfamily clone} and {\ttfamily finalize} are not accessible on arrays or interfaces, but all array types have a public version of {\ttfamily clone} which is accessible.

Determine whether this \doxyref{Object}{p.}{classjava_1_1lang_1_1_object} is semantically equal to another \doxyref{Object}{p.}{classjava_1_1lang_1_1_object}.

There are some fairly strict requirements on this method which subclasses must follow:\par
 
\begin{DoxyItemize}
\item It must be transitive. If {\ttfamily a.equals(b)} and {\ttfamily b.equals(c)}, then {\ttfamily a.equals(c)} must be true as well. 
\item It must be symmetric. {\ttfamily a.equals(b)} and {\ttfamily b.equals(a)} must have the same value. 
\item It must be reflexive. {\ttfamily a.equals(a)} must always be true. 
\item It must be consistent. Whichever value a.equals(b) returns on the first invocation must be the value returned on all later invocations. 
\item {\ttfamily a.equals(null)} must be false. 
\item It must be consistent with \doxyref{hashCode()}{p.}{classjava_1_1lang_1_1_object_a6c47262521fc94db1015b9866585e7e2}. That is, {\ttfamily a.equals(b)} must imply {\ttfamily a.hashCode() == b.hashCode()}. The reverse is not true; two objects that are not equal may have the same hashcode, but that has the potential to harm hashing performance. 
\end{DoxyItemize}

This is typically overridden to throw a \doxyref{ClassCastException}{p.}{classjava_1_1lang_1_1_class_cast_exception} if the argument is not comparable to the class performing the comparison, but that is not a requirement. It is legal for {\ttfamily a.equals(b)} to be true even though {\ttfamily a.getClass() != b.getClass()}. Also, it is typical to never cause a \doxyref{NullPointerException}{p.}{classjava_1_1lang_1_1_null_pointer_exception}.

In general, the Collections API (java.util) use the {\ttfamily equals} method rather than the {\ttfamily ==} operator to compare objects.

The default implementation returns {\ttfamily this == o}.

Called on an object by the Virtual Machine at most once, at some point after the \doxyref{Object}{p.}{classjava_1_1lang_1_1_object} is determined unreachable but before it is destroyed. You would think that this means it eventually is called on every \doxyref{Object}{p.}{classjava_1_1lang_1_1_object}, but this is not necessarily the case. If execution terminates abnormally, garbage collection does not always happen. Thus you cannot rely on this method to always work. For finer control over garbage collection, use references from the java.lang.ref package.

Virtual Machines are free to not call this method if they can determine that it does nothing important; for example, if your class extends \doxyref{Object}{p.}{classjava_1_1lang_1_1_object} and overrides finalize to do simply {\ttfamily super.finalize()}.

\doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} will be called by a \doxyref{Thread}{p.}{classjava_1_1lang_1_1_thread} that has no locks on any Objects, and may be called concurrently. There are no guarantees on the order in which multiple objects are finalized. This means that \doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} is usually unsuited for performing actions that must be thread-\/safe, and that your implementation must be use defensive programming if it is to always work.

If an \doxyref{Exception}{p.}{classjava_1_1lang_1_1_exception} is thrown from \doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} during garbage collection, it will be patently ignored and the \doxyref{Object}{p.}{classjava_1_1lang_1_1_object} will still be destroyed.

It is allowed, although not typical, for user code to call \doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} directly. User invocation does not affect whether automatic invocation will occur. It is also permitted, although not recommended, for a \doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} method to \char`\"{}revive\char`\"{} an object by making it reachable from normal code again.

Unlike constructors, \doxyref{finalize()}{p.}{classjava_1_1lang_1_1_object_ae7941b0c1ff37f29c70e470227463729} does not get called for an object's superclass unless the implementation specifically calls {\ttfamily super.finalize()}.
