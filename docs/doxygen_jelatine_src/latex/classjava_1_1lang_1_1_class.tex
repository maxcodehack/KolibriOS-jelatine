\subsection{Detailed Description}
A \doxyref{Class}{p.}{classjava_1_1lang_1_1_class} represents a Java type. There will never be multiple \doxyref{Class}{p.}{classjava_1_1lang_1_1_class} objects with identical names and ClassLoaders. Primitive types, array types, and void also have a \doxyref{Class}{p.}{classjava_1_1lang_1_1_class} object.

Arrays with identical type and number of dimensions share the same class. The array class ClassLoader is the same as the ClassLoader of the element type of the array (which can be null to indicate the bootstrap classloader). The name of an array class is {\ttfamily [$<$signature format$>$;}. 

For example, \doxyref{String}{p.}{classjava_1_1lang_1_1_string}[]'s class is {\ttfamily [Ljava.lang.String;}. boolean, byte, short, char, int, long, float and double have the \char`\"{}type name\char`\"{} of Z,B,S,C,I,J,F,D for the purposes of array classes. If it's a multidimensioned array, the same principle applies: {\ttfamily int[][][]} == {\ttfamily [[[I}.

There is no public constructor -\/ \doxyref{Class}{p.}{classjava_1_1lang_1_1_class} objects are obtained only through the virtual machine, as defined in ClassLoaders.
