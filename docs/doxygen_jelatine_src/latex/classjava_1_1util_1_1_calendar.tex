\subsection{Detailed Description}
This class is an abstract base class for Calendars, which can be used to convert between {\ttfamily \doxyref{Date}{p.}{classjava_1_1util_1_1_date}} objects and a set of integer fields which represent {\ttfamily YEAR}, {\ttfamily MONTH}, {\ttfamily DAY}, etc. The {\ttfamily \doxyref{Date}{p.}{classjava_1_1util_1_1_date}} object represents a time in milliseconds since the Epoch. \par


This class is locale sensitive. To get the Object matching the current locale you can use {\ttfamily getInstance}. You can even provide a locale or a timezone. {\ttfamily getInstance} returns currently a {\ttfamily GregorianCalendar} for the current date. \par


If you want to convert a date from the Year, Month, Day, DayOfWeek, etc. Representation to a {\ttfamily \doxyref{Date}{p.}{classjava_1_1util_1_1_date}}-\/Object, you can create a new \doxyref{Calendar}{p.}{classjava_1_1util_1_1_calendar} with {\ttfamily \doxyref{getInstance()}{p.}{classjava_1_1util_1_1_calendar_ac79d25925ab6a94a406c1a8549fae97f}}, {\ttfamily \doxyref{clear()}{p.}{classjava_1_1util_1_1_calendar_a8450eb2b4fbb9bba37e19a83051b9fee}} all fields, {\ttfamily \doxyref{set(int,int)}{p.}{classjava_1_1util_1_1_calendar_a898c50a2118c98c439ddcb8b0a30946d}} the fields you need and convert it with {\ttfamily \doxyref{getTime()}{p.}{classjava_1_1util_1_1_calendar_a12de1196e599b58132bbe34f37c2810f}}. \par


If you want to convert a {\ttfamily \doxyref{Date}{p.}{classjava_1_1util_1_1_date}}-\/object to the \doxyref{Calendar}{p.}{classjava_1_1util_1_1_calendar} representation, create a new \doxyref{Calendar}{p.}{classjava_1_1util_1_1_calendar}, assign the {\ttfamily \doxyref{Date}{p.}{classjava_1_1util_1_1_date}}-\/Object with {\ttfamily \doxyref{setTime()}{p.}{classjava_1_1util_1_1_calendar_afdaf61f294bdb0cf52324ebce06b25fd}}, and read the fields with {\ttfamily \doxyref{get(int)}{p.}{classjava_1_1util_1_1_calendar_af2dc13666d55ef74ab7bcece514d488d}}. \par


When computing the date from time fields, it may happen, that there are either two few fields set, or some fields are inconsistent. This cases will handled in a calendar specific way. Missing fields are replaced by the fields of the epoch: 1970 January 1 00:00. \par


To understand, how the day of year is computed out of the fields look at the following table. It is traversed from top to bottom, and for the first line all fields are set, that line is used to compute the day. \par



\begin{DoxyPre}month + day\_of\_month
month + week\_of\_month + day\_of\_week
month + day\_of\_week\_of\_month + day\_of\_week
day\_of\_year
day\_of\_week + week\_of\_year\end{DoxyPre}


The hour\_\-of\_\-day-\/field takes precedence over the ampm and hour\_\-of\_\-ampm fields. \par


{\bfseries Note:} This can differ for non-\/Gregorian calendar. \par


Other useful things you can do with an calendar, is {\ttfamily roll}ing fields (that means increase/decrease a specific field by one, propagating overflows), or {\ttfamily add}ing/substracting a fixed amount to a field.
