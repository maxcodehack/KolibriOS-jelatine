\subsection{Detailed Description}
{\ttfamily \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer}} represents a changeable {\ttfamily \doxyref{String}{p.}{classjava_1_1lang_1_1_string}}. It provides the operations required to modify the {\ttfamily \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer}}, including insert, replace, delete, append, and reverse. It is thread-\/safe; meaning that all modifications to a buffer are in synchronized methods.

{\ttfamily \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer}}s are variable-\/length in nature, so even if you initialize them to a certain size, they can still grow larger than that. {\itshape Capacity\/} indicates the number of characters the {\ttfamily \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer}} can have in it before it has to grow (growing the char array is an expensive operation involving {\ttfamily new}).

Incidentally, compilers often implement the \doxyref{String}{p.}{classjava_1_1lang_1_1_string} operator \char`\"{}+\char`\"{} by using a {\ttfamily \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer}} operation:\par
 {\ttfamily a + b}\par
 is the same as\par
 {\ttfamily new \doxyref{StringBuffer()}{p.}{classjava_1_1lang_1_1_string_buffer_a4c9ecbdda2c9417f42fcda31dea2ace1}.append(a).append(b).\doxyref{toString()}{p.}{classjava_1_1lang_1_1_string_buffer_aa4999ff69a5535cd0a281d1e20f1dcd5}}.

Classpath's \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer} is capable of sharing memory with Strings for efficiency. This will help when a \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer} is converted to a \doxyref{String}{p.}{classjava_1_1lang_1_1_string} and the \doxyref{StringBuffer}{p.}{classjava_1_1lang_1_1_string_buffer} is not changed after that (quite common when performing string concatenation).
