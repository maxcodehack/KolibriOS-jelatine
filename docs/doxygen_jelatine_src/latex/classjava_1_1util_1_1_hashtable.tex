\subsection{Detailed Description}
A class which implements a hashtable data structure. 

This implementation of \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} uses a hash-\/bucket approach. That is: linear probing and rehashing is avoided; instead, each hashed value maps to a simple linked-\/list which, in the best case, only has one node. Assuming a large enough table, low enough load factor, and / or well implemented hashCode() methods, \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} should provide O(1) insertion, deletion, and searching of keys. \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} is O(n) in the worst case for all of these (if all keys hash to the same bucket). 

This is a JDK-\/1.2 compliant implementation of \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable}. As such, it belongs, partially, to the Collections framework (in that it implements Map). For backwards compatibility, it inherits from the obsolete and utterly useless Dictionary class. 

Being a hybrid of old and new, \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} has methods which provide redundant capability, but with subtle and even crucial differences. For example, one can iterate over various aspects of a \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} with either an Iterator (which is the JDK-\/1.2 way of doing things) or with an \doxyref{Enumeration}{p.}{interfacejava_1_1util_1_1_enumeration}. The latter can end up in an undefined state if the \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} changes while the \doxyref{Enumeration}{p.}{interfacejava_1_1util_1_1_enumeration} is open. 

Unlike HashMap, \doxyref{Hashtable}{p.}{classjava_1_1util_1_1_hashtable} does not accept `null' as a key value. Also, all accesses are synchronized: in a single thread environment, this is expensive, but in a multi-\/thread environment, this saves you the effort of extra synchronization. However, the old-\/style enumerators are not synchronized, because they can lead to unspecified behavior even if they were synchronized. You have been warned. 

The iterators are {\itshape fail-\/fast\/}, meaning that any structural modification, except for {\ttfamily \doxyref{remove()}{p.}{classjava_1_1util_1_1_hashtable_a21678f63f9be4bc9fc11eaa1fe972c2c}} called on the iterator itself, cause the iterator to throw a {\ttfamily ConcurrentModificationException} rather than exhibit non-\/deterministic behavior.
