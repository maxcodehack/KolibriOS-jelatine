Parse the specified {\ttfamily \doxyref{String}{p.}{classjava_1_1lang_1_1_string}} as a {\ttfamily float}. The extended BNF grammar is as follows:\par
 
\begin{DoxyPre}
 {\itshape DecodableString\/}:
      ( [ {\ttfamily -} | {\ttfamily +} ] {\ttfamily NaN} )
    | ( [ {\ttfamily -} | {\ttfamily +} ] {\ttfamily Infinity} )
    | ( [ {\ttfamily -} | {\ttfamily +} ] {\itshape FloatingPoint\/}
              [ {\ttfamily f} | {\ttfamily F} | {\ttfamily d}
                | {\ttfamily D}] )
 {\itshape FloatingPoint\/}:
      ( \{ {\itshape Digit\/} \}+ [ {\ttfamily .} \{ {\itshape Digit\/} \} ]
              [ {\itshape Exponent\/} ] )
    | ( {\ttfamily .} \{ {\itshape Digit\/} \}+ [ {\itshape Exponent\/} ] )
 {\itshape Exponent\/}:
      ( ( {\ttfamily e} | {\ttfamily E} )
              [ {\ttfamily -} | {\ttfamily +} ] \{ {\itshape Digit\/} \}+ )
 {\itshape Digit\/}: {\itshape {\ttfamily '0'} through {\ttfamily '9'}\/}
 \end{DoxyPre}


NaN and infinity are special cases, to allow parsing of the output of toString. Otherwise, the result is determined by calculating {\itshape n $\ast$ 10$^{\mbox{exponent}}$ \/} to infinite precision, then rounding to the nearest float. Remember that many numbers cannot be precisely represented in floating point. In case of overflow, infinity is used, and in case of underflow, signed zero is used. Unlike \doxyref{Integer.parseInt}{p.}{classjava_1_1lang_1_1_integer_a369fccaa67bcd1e0f7d6695923628a65}, this does not accept Unicode digits outside the ASCII range.

If an unexpected character is found in the {\ttfamily \doxyref{String}{p.}{classjava_1_1lang_1_1_string}}, a {\ttfamily \doxyref{NumberFormatException}{p.}{classjava_1_1lang_1_1_number_format_exception}} will be thrown. Leading and trailing 'whitespace' is ignored via {\ttfamily \doxyref{String.trim()}{p.}{classjava_1_1lang_1_1_string_a43355a6a25f3c2504e45aad51d0e8ce9}}, but spaces internal to the actual number are not allowed.

To parse numbers according to another format, consider using java.text.NumberFormat.
