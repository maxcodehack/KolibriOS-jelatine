\subsection{Detailed Description}
A {\ttfamily \doxyref{NoClassDefFoundError}{p.}{classjava_1_1lang_1_1_no_class_def_found_error}} is thrown when a classloader or the Java Virtual Machine tries to load a class and no definition of the class can be found. This could happen when using the {\ttfamily new} expression or during a normal method call. The reason this would occur at runtime is because the missing class definition existed when the currently executing class was compiled, but now that definition cannot be found.
