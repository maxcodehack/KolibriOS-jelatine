\subsection{Detailed Description}
This class provides a mechanism for writing primitive Java datatypes to an {\ttfamily \doxyref{OutputStream}{p.}{classjava_1_1io_1_1_output_stream}} in a portable way. Data written to a stream using this class can be read back in using the {\ttfamily \doxyref{DataInputStream}{p.}{classjava_1_1io_1_1_data_input_stream}} class on any platform.

This method writes a Java {\ttfamily String} to the stream in a modified UTF-\/8 format. First, two bytes are written to the stream indicating the number of bytes to follow. Note that this is the number of bytes in the encoded {\ttfamily String} not the {\ttfamily String} length. Next come the encoded characters. Each character in the {\ttfamily String} is encoded as either one, two or three bytes. For characters in the range of {\ttfamily u0001} to u007F, one byte is used. The character value goes into bits 0-\/7 and bit eight is 0. For characters in the range of {\ttfamily u0080} to {\ttfamily u007FF}, two bytes are used. Bits 6-\/10 of the character value are encoded bits 0-\/4 of the first byte, with the high bytes having a value of \char`\"{}110\char`\"{}. Bits 0-\/5 of the character value are stored in bits 0-\/5 of the second byte, with the high bits set to \char`\"{}10\char`\"{}. This type of encoding is also done for the null character {\ttfamily u0000}. This eliminates any C style NUL character values in the output. All remaining characters are stored as three bytes. Bits 12-\/15 of the character value are stored in bits 0-\/3 of the first byte. The high bits of the first bytes are set to \char`\"{}1110\char`\"{}. Bits 6-\/11 of the character value are stored in bits 0-\/5 of the second byte. The high bits of the second byte are set to \char`\"{}10\char`\"{}.
